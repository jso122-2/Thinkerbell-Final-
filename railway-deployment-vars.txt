RAILWAY ENVIRONMENT VARIABLES FOR DEPLOYMENT
==============================================

Copy and paste these variables into Railway Dashboard:
Go to: Railway Dashboard → Your Service → Variables Tab

VARIABLE NAME: THINKERBELL_ENV
VALUE: production

VARIABLE NAME: PORT
VALUE: 8000

VARIABLE NAME: HOST
VALUE: 0.0.0.0

VARIABLE NAME: THINKERBELL_MODEL_DIR
VALUE: /app/models/optimum-model

VARIABLE NAME: PYTHONUNBUFFERED
VALUE: 1

VARIABLE NAME: PYTHONDONTWRITEBYTECODE
VALUE: 1

VARIABLE NAME: WEB_CONCURRENCY
VALUE: 1

==============================================
RAILWAY CLI ALTERNATIVE (if you prefer):
==============================================

railway variables set THINKERBELL_ENV production
railway variables set PORT 8000
railway variables set HOST 0.0.0.0
railway variables set THINKERBELL_MODEL_DIR /app/models/optimum-model
railway variables set PYTHONUNBUFFERED 1
railway variables set PYTHONDONTWRITEBYTECODE 1
railway variables set WEB_CONCURRENCY 1

==============================================
NOTES:
==============================================

1. In Railway Dashboard Variables Tab:
   - Enter VARIABLE NAME in the "Name" field
   - Enter VALUE in the "Value" field
   - DO NOT include the "=" sign - Railway handles this automatically

2. Required variables are the first 7 listed above
3. Optional variables can be added for enhanced security
4. Railway will automatically restart your service after adding variables
